## Project-Specific Context

### Architecture Overview
- **Backend**: Express.js server (`server.js`) with Socket.IO for real-time communication
- **Frontend**: React 18 + TypeScript SPA (`client/src/`)
- **Communication**: REST API + WebSocket for real-time updates
- **Data Storage**: JSON files (`config/`, `data/`) for configuration and positions
- **Hardware Layer**: Simulated motor controllers (MKS 57D/42D series)

### Key Technical Patterns

#### Real-time Communication
```javascript
// Always emit Socket.IO events for state changes
io.emit('robotPositionUpdate', newPosition);
io.emit('configurationUpdate', newConfig);
io.emit('gcodeStatus', { status: 'EXECUTING', progress: 50 });
```

#### Error Handling Pattern
```javascript
// Consistent API error responses
res.status(400).json({ 
  error: 'Validation failed', 
  details: validationErrors,
  timestamp: new Date().toISOString()
});
```

#### Frontend State Management
```typescript
// React hooks for Socket.IO integration
useEffect(() => {
  socket.on('robotPositionUpdate', handlePositionUpdate);
  return () => socket.off('robotPositionUpdate', handlePositionUpdate);
}, []);
```

### Critical File Locations
- **Main server**: `server.js` (Express routes, Socket.IO, hardware simulation)
- **Frontend app**: `client/src/App.tsx` (main UI with tabs)
- **Components**: `client/src/components/` (ManualControl, GCodeControl, PositionReplay, Configuration)
- **Styles**: `client/src/index.css` (all application styling)
- **Tests**: `test/` directory (Jest test files)
- **Hardware libs**: `lib/` (MKS motor controller simulation)
