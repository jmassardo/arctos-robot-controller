## Tips for Effective Task Assignment

### Be Specific

- Provide exact API endpoint names if you have preferences
- Specify UI placement and styling requirements
- Include specific test scenarios you want covered

### Include Context

- Mention related existing features that might be affected
- Reference specific files that definitely need modification
- Provide examples of similar implementations in the codebase

### Set Clear Success Criteria

- Define exactly what "working" means
- Specify performance requirements
- Include user experience expectations

### Consider Edge Cases

- Think about error conditions
- Consider concurrent usage scenarios
- Include hardware failure simulation requirements

Remember: The agent is designed to work independently and iterate until
completion. Provide comprehensive requirements upfront to minimize the need for
clarification.
